cern.pymad.io.tfs.tfsDict(inputfile)¶
tfs.tfsDict(inputfile)¶

Read a tfs table and returns table/summary info

The function takes in a tfs file. It will add all parameters into one dictionary, and the table into another dictionary.

Parameters:inputfile (string) – tfs file, full path
Raises ValueError:
 In case file path is not found
Return type:tuple containing dictionaries (tfs table , summary)

See also: pymad.domain.tfs

cern.pymad.tools.run.init(mode='cpymad', **kwargs)[source]¶

Initializes the environment for using pymad in one of the possible modes. The first argument determines the mode (allowed values are ‘cpymad’ and ‘jpymad’. If already a pymad service is running, then this function prints a warning and does nothing else.

Parameters:
  • mode (string) – Can be either ‘cpymad’ or ‘pymad’. This option determines which pymad-service will be initialized.
  • kwargs – All the rest of the arguments are directly passed to the constructor of the created PyMadService
Returns:

the newly created pymad service if one was created.
